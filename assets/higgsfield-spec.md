# Higgsfield Asset Generation Spec (v2 — environment overhaul)

Generation spec for the town's custom art. v2 supersedes the original 64x48
building pass: buildings are now 3-5x the player's width (real-town scale),
all doors face south, and every sprite must have an organic silhouette with
full transparency — no rectangle-shaped buildings, no background patches.

The map is generated by `scripts/gen_town_map.py` (committed); the footprint
table below is the shared contract between that script, `src/config.js`, and
the generated PNGs. Change a footprint → regenerate both the map and the art.

## Shared style note

Every asset — character, signpost, and all 7 buildings — must read as one
world, so every prompt ends with this exact suffix:

> top-down 3/4 perspective pixel art game asset for a 16-bit SNES-JRPG town,
> camera looking straight down at a slight forward tilt so ONLY the front
> face and the roof plane are visible, front wall vertical at the bottom of
> the frame, roof receding upward, strictly NO isometric or rotated-corner
> view, no side walls visible, warm late-afternoon sunlight from the
> upper-left (soft highlights on upper-left edges, soft cool shadows on
> lower-right edges), flat cel-shaded fills with 2-3 shade bands per surface,
> single consistent 1px dark outline (near-black #2b1d1f, never pure #000),
> muted saturated palette matching a small cozy town, no anti-aliasing, hard
> crisp pixel edges, a thin soft contact shadow hugging the bottom edge of
> the base so the asset sits grounded, solid uniform magenta #ff00ff
> background filling every pixel not covered by the asset, no text or UI
> elements in frame

Fixed for every asset: **front-face-and-roof view (never isometric)**,
**light upper-left**, **1px near-black outline**, **2-3 shade bands**,
**muted palette**, **contact shadow at the base** (new in v2 — buildings must
not float), **magenta #ff00ff backdrop** (Nano Banana Pro has no native
alpha; the backdrop is keyed out in post).

### Organic silhouettes (new in v2)

Buildings must NOT read as filled rectangles. Every building prompt should
include at least two silhouette-breaking elements — awnings, signage, steps,
props, chimneys, poles — that push outside the main wall line. The
chroma-key pass preserves whatever irregular outline the generation produces;
the in-engine footprint box only bounds the sprite, nothing fills it.

### Generation vs. target size

Generate at 2k resolution at the aspect ratio listed per asset, then in post
(`scripts/process_sprite.py`, committed):

1. chroma-key magenta → transparency (with fringe cleanup: pixels blending
   toward magenta at the silhouette edge are re-keyed or snapped to the
   outline color — check the result at 1:1 zoom, not just thumbnails)
2. autocrop to the content bounding box
3. downscale with **nearest-neighbor only** to fit the target footprint box
4. paste bottom-centered onto a transparent canvas of exactly the target size

## Footprint & door contract

Tile size 16px. The player is 32px (2 tiles) wide; buildings are 3-5x that.
`TownScene` anchors each sprite bottom-centered on its footprint rect and
never scales it — the PNG must be exactly the target size.

| Key | Project(s) | Tiles (W x H) | Target px | Gen aspect | Door(s) |
|---|---|---|---|---|---|
| `touchgrass_pavilion` | TouchGrass | 7 x 5 | 112 x 80 | 3:2 | open south side, center |
| `shortsleeve_dock` | Shortsleeve Travel | 8 x 6 | 128 x 96 | 4:3 | walk-on planks at west (left) edge |
| `twin_silo` | Terminal Dashboard + Blend Planner | 10 x 6 | 160 x 96 | 16:9 | TWO doors: one fully inside the left third, one fully inside the right third |
| `roberts_fuel` | Roberts Fuel Services | 8 x 5 | 128 x 80 | 3:2 | office door south, center-right |
| `summer_nicole_films` | Summer Nicole Films | 6 x 5 | 96 x 80 | 5:4 | storefront door south, center |
| `all_hands_detailing` | All Hands Detailing | 7 x 5 | 112 x 80 | 3:2 | open garage bay south, center |
| `fourwinds` | FourWinds | 6 x 5 | 96 x 80 | 5:4 | parlor door south, center |
| `signpost` | (town directory) | 2 x 3 | 32 x 48 | 2:3 | — |

All doors face SOUTH (toward the camera) — the map generator places each
trigger zone as a 3-tile-wide strip directly below the door position given
above. The twin silo's two doors are separate interactive zones in-engine
and must not blend into a single centered entrance.

File path convention: `assets/sprites/buildings/<key>.png`; the signpost is
`assets/sprites/signpost.png`.

## Character — player

Unchanged from v1 (32x32 frames, 160x128 sheet, 5 cols x 4 rows,
row = direction in the order down/left/right/up, col 0 idle, cols 1-4 walk).
The existing sheet is kept. See `src/anims.js` for the animation key
contract; frame index = `row * 5 + col`.

## Integration

`PreloadScene` requests every file above; missing files fail silently and
fall back to placeholders (generated flat sheet for the player, yellow
square for the signpost; a missing building PNG just leaves the ground
tiles visible — the footprint's static collision body exists either way).
Drop a finished PNG at the documented path and it is picked up on next load
with no code change. `TownScene` anchors building sprites bottom-centered on
their `BuildingFootprints` rect at native size (no scaling), so the PNG
dimensions must match the footprint table exactly.
